## Project Overview

A static personal dashboard hosted on GitHub Pages. Three tabs: Dimoo figurine collection, movie reviews, and restaurant reviews. No build system — runs directly in the browser.

## Running Locally

```bash
python3 -m http.server 8000
# open http://localhost:8000/index.html
```

A local server is required because `index.html` fetches JSON files via `fetch()`.

## Syncing Data from Obsidian

Movie and restaurant data lives in an Obsidian vault at:
`~/Library/Mobile Documents/iCloud~md~obsidian/Documents/personal`

Run the sync script whenever Obsidian notes are updated:
```bash
python3 sync_from_obsidian.py
```

This reads `cards/movies/` (organized by year) and `cards/restaurants/` from the vault, parses Dataview inline frontmatter (`key:: value` format at the top of each file), and writes `movies.json` and `restaurants.json`.

## Architecture

Single `index.html` with all CSS, HTML, and JavaScript inline. Three JSON files are the data sources — edit via Obsidian then run the sync script.

```
index.html                — full app (tabs, charts via Chart.js CDN, tables)
sync_from_obsidian.py     — pulls data from Obsidian vault → movies.json / restaurants.json
dimoos.json               — Dimoo collection data (edited manually)
movies.json               — generated by sync script (98 entries)
restaurants.json          — generated by sync script (63 entries)
collections.db            — SQLite DB (not used by the web interface)
dimoos.html               — legacy single-page viewer (superseded by index.html)
```

## JavaScript Structure

```
init()
  ├─ fetchJSON() × 3 in parallel — graceful 404 returns []
  ├─ setupTabs() — lazy chart init (charts only render when tab is first opened)
  ├─ setupFilters() — debounced (200ms) table re-renders
  ├─ renderDimoo() — called immediately on load
  ├─ renderMovies() — lazy, called on first tab visit
  └─ renderRestaurants() — lazy, called on first tab visit
```

Chart instances are stored in `CI` map. `safeChart()` destroys the old instance before creating a new one (prevents "canvas already in use" errors). All user data is rendered via DOM `textContent` — never `innerHTML` with data values.

## Data Schemas

### `dimoos.json` (edited manually)
```json
{ "name": "calm down", "series": "by your side", "number": "1",
  "owned": "yes", "how": "blind box", "who": "self bought",
  "purchase_date": "08/30/2024" }
```
- `owned`: `"yes"` or `"no"`
- `how`: blind box | pre-owned | gift - [name] | trade | self bought | n/a
- `purchase_date`: MM/DD/YYYY or `"n/a"`

### `movies.json` (generated by sync script)
```json
{ "title": "Kinda Pregnant", "year": 2025, "watch_date": "2025-02-08",
  "location": "daly city home", "people": "alone", "rating": 7,
  "overall": "funny movie but gave me second hand embarrassment" }
```
- `rating`: integer 1–10
- `watch_date`: YYYY-MM-DD (ISO 8601)
- Obsidian source fields: `movie_name`, `watch_date`, `location`, `people`, `rating`, `overall`

### `restaurants.json` (generated by sync script)
```json
{ "name": "ushi taro ramen sunset", "date": "2025-09-27",
  "location": "sunset sf", "people": "eric and jordan mai",
  "would_return": "yes", "food": "S", "service": "B",
  "atmosphere": "B", "value": "B", "overall": "A" }
```
- `would_return`: `"yes"`, `"no"`, or `"maybe"`
- Grade fields (`food`, `service`, `atmosphere`, `value`, `overall`): S | A | B | C | D with optional +/- modifiers (e.g. `"B+"`, `"C-"`)
- `date`: YYYY-MM-DD (ISO 8601)
- Obsidian source fields: `restaurant_name`, `date`, `location`, `people`, `would_return`, `food`, `service`, `atmosphere`, `value`, `overall`

## Key Implementation Notes

- **Date formats**: `dimoos.json` uses MM/DD/YYYY (legacy); movies and restaurants use ISO 8601. The `parseDate()` helper handles both.
- **Grade extraction**: `gradeBase(g)` extracts the first character of a grade string (so `"B+"` → `"B"`) for charting and filtering.
- **Location normalization**: `normalizeWatchLocation()` maps movie `location` strings to `"Theatre"` or `"Home"`.
- **Chart lazy init**: Charts only render after their tab is first clicked. Chart.js requires the canvas to be visible (non-zero dimensions) when initialized.
